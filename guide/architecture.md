# アーキテクチャ概要 / Architecture Overview

> 公開リポジトリに含まれるコードの読み方ガイド。実装の入口はこのページから。

## モジュール構成

```
Source/
├── PluginProcessor.h/cpp   … DSP ホスト。パラメーター(APVTS)管理・16 ボイスの
│                             juce::Synthesiser・スペクトラム用ロックフリー FIFO
├── PluginEditor.h/cpp      … UI ホスト（4 モード切替・900×580）
├── ParameterMapper.h       … AI 解析結果 → パラメーター変換（ステートレス）
├── be/                     … バックエンド（DSP・解析・プリセット）
│   ├── SynthVoice.h        … 1 ボイス: 3×OSC + ユニゾン + ピッチ EG + アンプ ADSR + IIR LPF
│   ├── AudioAnalyzer.h/cpp … AI マッチ用の FFT + ピッチ解析（バックグラウンドスレッド）
│   ├── PresetManager.h     … プリセット保存/読込・旧フォーマット移行
│   └── dsp/
│       ├── FMEngine.h      … 4-OP FM（接続行列・フィードバック・オーバーサンプリング）
│       ├── NoiseGenerator.h… White/Pink/Brown（xorshift128+・インスタンス毎シード）
│       └── WaveTable.h     … サイン波テーブル（線形補間）
├── fe/                     … フロントエンド（UI コンポーネント）
└── render/OfflineRenderer.cpp … ヘッドレス preset→WAV レンダラー（RdhRender・CI スモーク兼用）
```

## シグナルフロー（1 ボイス）

```
[OSC 1-3 × AmpEnv]     ┐
[FM Engine（独自EG）]   ├──→ [Mixer] → [IIR LPF] → 出力へ加算
[Noise（FilterSend）]  ┘
[Noise（DirectOut）] ────────────────────────→ 出力へ加算（フィルターバイパス）
```

- FM とノイズはそれぞれ**独自のエンベロープ**を持ち、アンプエンベロープの影響を受けません
  （FM セクションは独立モジュールとして設計）。
- ボイスは全エンベロープが終わるまで生存します（長いリリースが途切れない）。
- マスターボリュームは平滑化（クリック防止）してから適用されます。

## 設計原則

1. **リアルタイム安全**: オーディオスレッドでのメモリ確保・ロック・ファイル I/O 禁止。
   GUI⇄DSP はロックフリー FIFO・アトミックで通信。
2. **複数インスタンス独立**: 可変グローバル状態なし。ノイズの乱数はインスタンス毎に独立シード。
3. **後方互換**: 旧バージョンのプリセットは新パラメーターをデフォルト値で補完して読み込む。
   パラメーター ID は追加のみ（削除・変更しない）。
4. **CPU は計測で管理**: 最適化・機能追加の判断はヘッドレスレンダラー等の計測を根拠に行う。

## 検証の考え方

- `RdhRender`（`render/OfflineRenderer.cpp`）で**同一プリセットを決定論的に描画**でき、
  変更前後の PCM 比較（ビット一致）・スペクトラム解析・CPU ベンチが GUI なしで行えます。
- CI はこのレンダラーでスモークテスト（描画できて無音でない）を行います。
- 乱数を使うノイズ系プリセットはインスタンス毎シードのため**ビット一致比較の対象外**です
  （設計どおりの挙動）。
